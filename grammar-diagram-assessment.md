## 高優先級（建議優先加入圖表）

### 1. 格助詞系統圖表（091_case_particles.md）
**優先級**: Critical
**圖表類型**: 階層表格圖 + 選擇流程圖
**原因**:
- 格助詞是日文基礎核心，8個核心助詞的選擇規則複雜
- 學習者最常混淆「に」vs「で」、「を」vs「が」
- 視覺化可大幅降低學習難度

**建議圖表內容**:
```
【格助詞選擇流程圖】
動詞是什麼？
├─ 存在動詞（ある/いる）→ 場所用「に」
├─ 移動動詞（行く/來る）→ 目的地用「に」或「へ」
├─ 他動詞 → 賓語用「を」
└─ 動作動詞 → 場所用「で」、手段用「で」
```

### 2. 自他動詞判斷系統（092_transitive_intransitive.md）
**優先級**: Critical
**圖表類型**: 判斷樹 + 助詞對比表
**原因**:
- 自他動詞區別是N4核心概念
- 助詞判斷法（を→他動詞，が→自動詞）需要視覺化
- 視角轉換（主動 vs 狀態變化）需要清晰對比

**建議圖表內容**:
```
【自他動詞判斷樹】
看助詞！
├─ を + 動詞 → 他動詞（ドアを開ける）
└─ が + 動詞 → 自動詞（ドアが開く）

看意圖！
├─ 有意圖的動作 → 他動詞
└─ 自然發生的變化 → 自動詞
```

### 3. 生命性判斷系統（096_aru_iru.md）
**優先級**: High
**圖表類型**: 判斷樹 + 分類表
**原因**:
- N5最基礎的概念，但學習者常犯錯
- 「能否自主移動」的判斷標準需要視覺化
- 植物的特例（有生命但用ある）需要強調

**建議圖表內容**:
```
【生命性判斷樹】
它能自己走動嗎？
├─ 能 → いる（人、動物、昆蟲）
└─ 不能 → ある（物品、植物、抽象概念）

【特別注意】
植物：雖然是生物，但不能自主移動 → ある
機器人：雖然能移動，但不是生物 → ある
```

### 4. 敬語系統架構（005_keigo_system.md）
**優先級**: High
**圖表類型**: 三層階層圖 + 內外關係表
**原因**:
- 敬語是日語的核心特色，系統複雜
- 尊敬/謙讓/丁寧三種系統需要清晰區分
- 內外關係的判斷（對外說內部人士用謙讓語）需要視覺化

**建議圖表內容**:
```
【敬語選擇流程】
誰的動作？
├─ 對方/上級的動作 → 尊敬語（召し上がる、お〜になる）
├─ 自己/下級的動作 → 謙讓語（いただく、お〜する）
└─ 一般禮貌 → 丁寧語（です・ます）

【內外關係】（對外說話時）
├─ 自己公司的人（包括社長）→ 謙讓語
└─ 對方公司的人 → 尊敬語
```

### 5. 〜ている的雙重意義（075_te_iru.md）
**優先級**: High
**圖表類型**: 語義分支圖
**原因**:
- N5最重要文法之一，但語義複雜
- 動詞類型決定意義（持續動詞→進行，瞬間動詞→狀態）
- 視覺化可幫助理解判斷標準

**建議圖表內容**:
```
【〜ている的意義判斷】
動詞類型？
├─ 持續動作動詞（食べる、走る）→ 進行式「正在...」
└─ 瞬間動詞（知る、住む、結婚する）→ 結果狀態「...著」

【特殊動詞】
知る → 知っている（只能是狀態「知道」，不能是進行）
移動動詞 → 行っている（已經去了，在那裡）
```

### 6. 引用表現的時態調整（094_quotation_expression.md）

## 不建議加入圖表的原因分析

以下類型的卡片**不需要圖表**，因為：

1. **單一規則型**（如078-081, 082-084, 086-089）
   - 形成規則簡單（た形+ことがある、ます→ました）
   - 無複雜判斷或多選項
   - 文字說明已清楚

2. **基礎變化型**（如082_mashita, N5級別）
   - 動詞變化規則簡單直接
   - 學習者只需記憶，不需判斷

3. **列舉型**（如095_conjunction_particles）
   - 接續助詞的列舉，表格已足夠
   - 無需視覺化流程

4. **文法概念型**（如085_rentai_shushoku）
   - 連體修飾是語言學概念
   - 文字說明比圖表更清晰
