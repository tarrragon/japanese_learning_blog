助詞「も」所承載的「添加」概念，是日語資訊結構中用於建立事物間共同性的核心語用機制。

## 日文

も（添加・類同の概念）

### 概念情報

| 項目 | 內容 |
|------|------|
| 概念類型 | 語用概念・資訊結構 |
| 核心功能 | 添加・類同・並列 |
| 英語對應 | also, too, as well, both...and |
| 相關助詞 | も、さえ、まで、すら |

## 日文解釋

「添加」（addition）とは、既知の情報に対して、同じ性質や状態を持つ新たな情報を付け加える語用論的操作である。日本語において、この概念は副助詞「も」によって最も典型的に表現される。

「も」の根本的な機能は、「前提となる情報の存在」と「新情報の類似性」を同時に示すことにある。例えば「田中さんも来た」という発話は、「誰かが来た」という前提と、「田中さんも同様に来た」という新情報を伝えている。

この添加の概念は、対比（contrast）や焦点（focus）といった他の情報構造の概念と密接に関連している。特に「も」は、主題助詞「は」との対照において、「排他的対比」ではなく「包括的並列」を表すという特徴がある。

また、添加の概念は程度副詞や数量表現と結びつくことで、「予想を超える」というニュアンスを生み出す。「3時間も待った」は単なる添加ではなく、「3時間という長さ」に対する話者の驚きや不満を含意する。

## 英文解釋

The concept of "addition" in Japanese pragmatics refers to the communicative function of presenting new information as sharing the same property or state as previously established information. The particle "mo" serves as the primary marker for this concept.

The fundamental pragmatic structure of addition involves two components: (1) a presupposition that something with a certain property exists, and (2) an assertion that the focused element shares that property. When someone says "Tanaka-san mo kita" (Tanaka also came), they presuppose that someone came and assert that Tanaka did likewise.

This differs from topic marking with "wa" in that "mo" creates an inclusive parallel rather than an exclusive contrast. While "wa" can imply "this one, but not others," "mo" explicitly signals "this one, in addition to others."

The additive function extends beyond simple listing to express speaker attitude when combined with quantity expressions. "San-jikan mo matta" (waited for as long as three hours) conveys not just addition but the speaker's subjective evaluation that the quantity exceeds expectations.

## 中文解釋

「添加」是日語語用學中的核心概念，指的是將新資訊呈現為與已知資訊具有相同性質的溝通功能。助詞「も」是這個概念最典型的表達方式。

添加概念的語用結構包含兩個要素：（1）存在某個具有特定性質的事物（前提），（2）被聚焦的新元素也具有相同性質（斷言）。例如說「田中さんも来た」時，預設了「有人來了」，並斷言「田中也同樣來了」。

這與主題助詞「は」的標記方式不同。「は」可能暗示「這個，但不是其他的」（排他性對比），而「も」則明確表示「這個，加上其他的」（包容性並列）。

當添加功能與數量表現結合時，會產生「超出預期」的語感。「3時間も待った」不只是陳述等了三小時，更傳達了說話者認為三小時「太長了」的主觀評價。這種用法展現了日語如何透過簡單的助詞來編碼說話者的態度和情感。

## 核心用法

### 用法 1：建立事物間的共同性

當說話者想要表達某事物與已知事物具有相同性質時使用。這是「も」最基本的添加功能。

**例句 1**
```
A: 私はコーヒーが好きです。
B: 私もコーヒーが好きです。
A: I like coffee.
B: I also like coffee.
A: 我喜歡咖啡。
B: 我也喜歡咖啡。
```

**例句 2**
```
東京は暑い。大阪も暑い。
Tokyo is hot. Osaka is also hot.
東京很熱。大阪也很熱。
```

### 用法 2：並列多個同質項目

使用「AもBも」結構，將多個項目作為同質的整體呈現。強調所有項目都具有相同的性質。

**例句 3**
```
日本語も英語も話せます。
I can speak both Japanese and English.
日語和英語我都會說。
```

### 用法 3：表達超出預期的程度

與數量詞結合，傳達說話者對數量或程度的主觀評價，通常帶有驚訝、不滿或感嘆的語氣。

**例句 4**
```
あの映画は3回も見ました。
I watched that movie as many as three times.
那部電影我竟然看了三次。
```

### 用法 4：全稱量化（與疑問詞結合）

與疑問詞結合形成「誰も」「何も」「どこも」等表現，在肯定句中表示「全部都」，在否定句中表示「全部都不」。

**例句 5**
```
今日は誰も来なかった。
No one came today.
今天誰都沒來。
```

## 概念辨析

### 「も」與「は」的資訊結構差異

|  | も（添加） | は（主題/對比） |
|------|------|------|
| **前提** | 存在同類項目 | 不一定 |
| **斷言** | 包容性（也是） | 可能是排他性（這個是，其他不是） |
| **焦點** | 所標記的項目 | 述語部分 |
| **隱含意義** | 「除此之外還有」 | 「就這個而言」 |

### 「も」與「さえ」的程度差異

|  | も | さえ |
|------|------|------|
| **添加性** | 一般添加 | 極端案例的添加 |
| **說話者態度** | 中立或輕微強調 | 強烈驚訝或強調 |
| **例句** | 子供**も**知っている（小孩也知道） | 子供**さえ**知っている（連小孩都知道） |

## 跨語言視角

### 與英語 "also/too" 的異同

英語的 "also" 和 "too" 同樣表達添加概念，但位置較為固定：
- "also" 通常出現在動詞前或句首
- "too" 通常出現在句尾

日語的「も」則直接附著在要強調的元素後面，因此能更精確地標示添加的對象：
- 私**も**行く（我**也**去）← 強調主體
- 京都**にも**行く（**也**去京都）← 強調目的地

### 與中文「也」的對應

中文的「也」在功能上與「も」高度對應，但「も」能夠與數量詞結合產生「超出預期」的意義，這在中文中需要用「竟然」「居然」等詞來表達：
- 3時間**も**待った → 竟然等了三小時

## 常見錯誤

### 錯誤 1：混淆「も」的焦點對象

❌ 誤：（想說「我也去大阪」）私は大阪にも行きます。
✅ 正：私も大阪に行きます。
說明：「にも」強調的是「大阪」這個地點（也去大阪，不只去其他地方），而「私も」強調的是「我」這個主體（我也去，不只別人去）。

### 錯誤 2：並列結構不完整

❌ 誤：りんごも、バナナが好きです。
✅ 正：りんごも、バナナも好きです。
說明：「AもBも」結構中，所有並列項目都必須加「も」，不能混用「も」和「が」。

## 學習要點

1. **前提意識**：使用「も」時，必須存在「與之相同的其他事物」這個前提。無中生有地使用「も」會造成困惑。
2. **位置決定焦點**：「も」附著在哪個詞後面，就是在強調那個詞與其他事物相同。
3. **主觀評價功能**：「數量詞＋も」不只是添加，更傳達說話者「多於預期」的評價。
4. **與「は」的選擇**：當要表達「也是如此」時用「も」，當要表達「就此而言」或進行對比時用「は」。
5. **疑問詞結合規則**：「疑問詞＋も」的意義取決於句子是肯定還是否定。
