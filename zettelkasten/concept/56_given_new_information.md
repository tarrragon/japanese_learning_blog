# 舊資訊與新資訊（Given-New Information）

## 核心概念

### 日文解釋
文中で、聞き手がすでに知っている情報を「旧情報（きゅうじょうほう）」、まだ知らない新しい情報を「新情報（しんじょうほう）」と呼ぶ。日本語では、「は」が旧情報を、「が」が新情報を示すことが多い。

### 繁體中文解釋
在文章中，聽者已經知道的資訊稱為「舊資訊」，尚未知道的新資訊稱為「新資訊」。在日文中，「は」通常標示舊資訊，「が」通常標示新資訊。

### English Explanation
In discourse, information the listener already knows is called "given information," while new information they don't yet know is called "new information." In Japanese, は typically marks given information, while が typically marks new information.

---

## 資訊結構理論

### 基本原則

**舊資訊先行原則**：
- 句子通常從已知資訊開始
- 以已知為基礎引入新資訊
- 維持語境連貫性

**標記方式**：
- **は**：標示舊資訊（話題、已知事物）
- **が**：標示新資訊（焦點、新資訊）

### 語境依賴性

資訊的新舊不是絕對的，而是相對於語境：

```
A：「誰が来ましたか？」（誰來了？）
B：「田中さんが来ました。」（田中先生來了。）
   └→ 田中さん = 新資訊（回答 who）

A：「田中さんは何をしましたか？」（田中先生做了什麼？）
B：「田中さんは本を読みました。」（田中先生讀了書。）
   └→ 田中さん = 舊資訊（已在問句提及）
```

## は與が的資訊結構對比

### 舊資訊 + は

**用法**：話題設定、已知事物描述

```
私は学生です。
└→ 我 = 舊資訊（說話者自己）
```

```
東京は大きい都市です。
└→ 東京 = 舊資訊（共同知識）
```

### 新資訊 + が

**用法**：新事物引入、焦點標示

```
雨が降っています。
└→ 雨 = 新資訊（新出現的事件）
```

```
誰が来ましたか？ — 田中さんが来ました。
└→ 田中さん = 新資訊（回答疑問詞）
```

## 例句分析

### 例句 1：舊資訊標示
```
太郎は昨日学校に行きました。今日も学校に行きます。
（太郎昨天去了學校。今天也去學校。）
```

**分析**：
- 第二句的「今日も」= 舊資訊（延續第一句的主題）
- 使用 も（也）強調舊資訊的延續

### 例句 2：新資訊引入
```
外で音がします。誰かが来たようです。
（外面有聲音。好像有人來了。）
```

**分析**：
- 音 = 新資訊（新察覺的事件）
- 誰か = 新資訊（推測的新事物）

### 例句 3：資訊轉換
```
Q: 犯人は誰ですか？
A: 犯人は田中です。
   └→ 犯人 = 舊資訊（問句中提及）
   └→ 田中 = 新資訊（回答內容）
```

**分析**：
- 犯人 用 は（問句已提及，為舊資訊）
- 田中 為述語（提供新資訊）

## 常見語境

### 對話應答
```
Q: 何が起きましたか？
A: 事故が起きました。
   └→ 事故 = 新資訊
```

### 故事敘述
```
昔々、ある所におじいさんとおばあさんがいました。
おじいさんは山へ芝刈りに、おばあさんは川へ洗濯に行きました。
└→ 第二句的おじいさん・おばあさん = 舊資訊（第一句已引入）
```

### 描述性語句
```
春になると、桜が咲きます。
└→ 桜 = 新資訊（春季新出現的事物）
```

---

## 學習重點

1. **資訊新舊是相對的**：取決於語境和聽者的知識狀態
2. **は vs が 的選擇**：基於資訊結構，不僅是文法規則
3. **對話連貫性**：舊資訊先行，新資訊後置
4. **疑問詞應答**：疑問詞所問的部分為新資訊，用 が

---

## 常見錯誤

❌ **忽略語境脈絡**
```
誤：雨は降っています。（初次提及雨）
正：雨が降っています。
```

❌ **疑問詞應答錯誤**
```
誤：誰は来ましたか？
正：誰が来ましたか？
```

❌ **已知資訊用が**
```
誤：田中さんが何をしましたか？（田中已在語境中）
正：田中さんは何をしましたか？
```
