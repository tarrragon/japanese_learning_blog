## 日文

できる

**羅馬拼音**：dekiru

**文法類型**：可能形動詞

### 文法情報

| 項目 | 內容 |
|------|------|
| 文法類型 | 可能形（する動詞的特殊形式） |
| JLPT | N5 |
| 使用場面 | 日常、正式、商務 |
| 形成 | する → できる |

### 形成規則

**基本形式**
```
する → できる
```

**範例**
```
勉強する → 勉強できる
運転する → 運転できる
料理する → 料理できる
```

**漢字表記**
- 出来る（較正式，書面語）
- できる（較常見，口語）

## 日文解釋

「できる」は、「する」の可能形[^kanou-kei]として機能する特殊な動詞です。「できる」は他の動詞と異なり、独立した一段動詞[^ichidan]として活用します。

「する」から派生した「できる」は、主に三つの用法があります：

1. **能力の表現**：何かをする能力や技能があることを示します。例えば、「日本語ができる」は日本語を話す・読む・書く能力があることを意味します。

2. **可能性の表現**：何かが実現可能であることを示します。「明日行けばできる」のように、条件が整えば実現可能という意味です。

3. **完成・出現の表現**：何かが完成する、または出現することを示します。「建物ができた」は建物が完成したことを意味します。

助詞の使い方[^joshi-usage]にも注意が必要です。通常の「する」動詞は「を」を取りますが、可能形[^kanou-kei]の「できる」は「が」を取ります。これは日本語の文法規則[^grammar-rule]の重要な特徴です。

## 英文解釋

"Dekiru" functions as the potential form[^kanou-kei] of the verb "suru" (to do), but unlike other potential forms, it is an independent ichidan verb[^ichidan] with its own conjugation pattern.

This verb has three primary uses:

1. **Expressing ability**: It indicates the capability or skill to do something. For example, "Nihongo ga dekiru" means having the ability to speak, read, or write Japanese.

2. **Expressing possibility**: It shows that something is feasible or achievable. When you say "Ashita ikeba dekiru," it means it's possible if you go tomorrow.

3. **Expressing completion or emergence**: It indicates that something has been completed or has come into existence. "Tatemono ga dekita" means a building has been completed.

An important grammatical point[^grammar-rule] is the particle usage[^joshi-usage]: while "suru" takes the particle "wo," the potential form[^kanou-kei] "dekiru" takes the particle "ga." This is a fundamental characteristic of Japanese potential expressions.

## 中文解釋

「できる」是「する」的可能形[^kanou-kei]，但與其他動詞的可能形不同，它是一個獨立的一段動詞[^ichidan]，有自己的活用模式。

這個動詞主要有三種用法：

1. **表達能力**：表示具有做某事的能力或技能。例如「日本語ができる」意思是具備說、讀、寫日文的能力。

2. **表達可能性**：表示某事是可行或可實現的。「明日行けばできる」意思是如果明天去就可以做到。

3. **表達完成或出現**：表示某事已經完成或出現。「建物ができた」意思是建築物已經完工。

需要注意助詞用法[^joshi-usage]：一般的「する」動詞搭配助詞「を」，但可能形[^kanou-kei]的「できる」要搭配助詞「が」。這是日文文法規則[^grammar-rule]的重要特徵。

## 核心用法

### 用法 1：表達能力或技能

表示具備某種能力、技能或知識。

**例句 1**
```
私は日本語ができます。
I can speak Japanese. / I know Japanese.
我會日文。
```

**例句 2**
```
田中さんはピアノができる。
Tanaka-san can play the piano.
田中先生會彈鋼琴。
```

### 用法 2：表達可能性或可行性

表示某事在某種條件下是可能實現的。

**例句 3**
```
明日までにレポートができますか。
Can you finish the report by tomorrow?
明天之前能完成報告嗎？
```

**例句 4**
```
この方法ならできると思います。
I think it's possible with this method.
我認為用這個方法可以做到。
```

### 用法 3：表達完成或出現

表示某物已經完成、建成或出現。

**例句 5**
```
新しい駅ができました。
A new station has been built.
新車站建好了。
```

## 文法規則

### 規則 1：助詞搭配差異

**する → を（動作的對象）**
```
勉強をする（做學習這個動作）
料理を作る（做料理）
```

**できる → が（能力或可能性的對象）**
```
勉強ができる（能夠學習）
料理ができる（會做料理）
```

### 規則 2：活用方式

「できる」作為一段動詞的活用：

| 活用形 | 形式 | 例句 |
|--------|------|------|
| 辞書形 | できる | 日本語ができる |
| ます形 | できます | できますか |
| て形 | できて | できてよかった |
| た形 | できた | ついにできた |
| ない形 | できない | できないことはない |
| 可能形 | できられる（不使用） | - |

**注意**：「できる」本身就是可能形，不再有可能形變化。

### 規則 3：否定形式的含義

**できない**
```
私は泳ぐことができない。
I cannot swim.
我不會游泳。
```

表示沒有能力或不可能實現。

## 常見錯誤

### 錯誤 1：助詞使用錯誤

❌ 誤：日本語をできます。
✅ 正：日本語ができます。

說明：「できる」是可能形，要用「が」而不是「を」。這是因為可能形表達的是能力狀態，而非動作對象。

### 錯誤 2：混淆する和できる

❌ 誤：明日勉強をできます。
✅ 正：明日勉強ができます。/ 明日勉強します。

說明：「する」表示實際做某動作，「できる」表示有能力做或可能做到。不要混用助詞「を」和「が」。

### 錯誤 3：過度使用漢字

❌ 誤：出来ますか。（過於正式）
✅ 正：できますか。（自然口語）

說明：日常會話中通常使用平假名「できる」，漢字「出来る」多用於正式書面語。

## 學習要點

1. **獨立動詞**：「できる」不只是可能形，而是獨立的一段動詞，有完整的活用系統。

2. **助詞搭配**：記住「する＋を」vs「できる＋が」的區別，這是初學者最常犯的錯誤。

3. **三種含義**：能力、可能性、完成，需要根據語境判斷具體含義。

4. **書面與口語**：「できる」（口語）比「出来る」（書面）更常用，除非是正式文件。

5. **否定表達**：「できない」清楚表達不可能或沒有能力，是常用的否定表達方式。

## 註解

[^kanou-kei]: **可能形** - 表示能力或可能性的動詞形式。一般動詞的可能形是在動詞語幹後加「られる」（一段動詞）或「える」（五段動詞），但「する」的可能形是特殊的「できる」。詳見 [可能形](potential_form.md)

[^ichidan]: **一段動詞** - 語幹不變，只變化「る」的動詞類型。「できる」的活用：できる → できます、できて、できた、できない。詳見 [一段動詞](grammar/verb_types/ichidan.md)

[^joshi-usage]: **助詞用法** - 日文助詞標示名詞與動詞的關係。「を」標示動作對象（勉強をする），「が」標示能力對象或主語（勉強ができる）。詳見 [助詞總覽](../particle/overview.md)

[^grammar-rule]: **文法規則** - 日文可能形一般使用「が」而非「を」，這是因為可能形表達的是狀態（能夠～）而非動作（做～）。這個規則適用於所有可能形動詞。詳見 [可能形助詞規則](potential_particles.md)
