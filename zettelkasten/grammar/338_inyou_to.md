引用助詞「と」是日語中最基本的引用標記，連接被引用的內容與引用動詞，形成引用表現的核心結構。

<!--more-->

## 日文

引用表現：と（いんようひょうげん：と）

### 文法情報

| 項目 | 內容 |
|------|------|
| 文法類型 | 引用助詞 |
| JLPT | N4 |
| 使用場面 | 正式、日常 |
| 形成 | 引用內容 + と + 引用動詞 |

### 形成規則

**基本形式**
```
「直接引用」+ と + 言う/思う/聞く
普通形 + と + 言う/思う/聞く
```

**範例**
```
「行きます」と言う → 直接引用
行くと言う → 間接引用
何と言いましたか → 疑問詞との組み合わせ
```

## 日文解釋

引用助詞「と」は、発言・思考・伝聞などの内容を示すマーカーとして機能する格助詞である。「と」は被引用部分（引用される内容）と引用動詞（言う・思う・聞くなど）を繋ぐ役割を果たす。

**「と」の本質的機能**：

1. **境界標示**：引用される内容と引用する行為を区切る
2. **内容指示**：「と」の直前に来る部分が引用内容であることを示す
3. **形式保存**：直接引用の場合、原話の形式をそのまま保存する

**引用動詞との組み合わせ**：

引用の「と」は必ず引用動詞を必要とする。主な引用動詞には以下のものがある：

- **発言系**：言う、話す、答える、尋ねる、叫ぶ、呼ぶ
- **思考系**：思う、考える、感じる、信じる、決める
- **伝達系**：聞く、読む、書く、伝える、知らせる
- **命名系**：呼ぶ、名付ける、いう（〜という名前）

「と」は条件を表す用法（「雨が降ると中止だ」）と形が同じだが、後続する動詞の種類によって区別される。引用の「と」は必ず発言・思考・伝達に関する動詞が続く。

## 英文解釋

The quotation particle "to" functions as a case particle that marks the content of speech, thought, or hearsay. It connects the quoted portion (content being quoted) with quotation verbs (such as iu, omou, kiku).

**Essential functions of "to"**:

1. **Boundary marking**: Separates quoted content from the quoting action
2. **Content indication**: Signals that what precedes "to" is the quoted content
3. **Form preservation**: In direct quotation, preserves the original form of speech

**Combination with quotation verbs**:

Quotative "to" always requires a quotation verb. Main categories include:

- **Speech verbs**: iu (say), hanasu (speak), kotaeru (answer), tazuneru (ask), sakebu (shout), yobu (call)
- **Thought verbs**: omou (think), kangaeru (consider), kanjiru (feel), shinjiru (believe), kimeru (decide)
- **Communication verbs**: kiku (hear), yomu (read), kaku (write), tsutaeru (convey), shiraseru (inform)
- **Naming verbs**: yobu (call), nazukeru (name), iu (〜toiu name)

The quotative "to" shares the same form as the conditional "to" (as in "ame ga furu to chuushi da"), but they are distinguished by the type of verb that follows.

## 中文解釋

引用助詞「と」是一個格助詞，用來標示發言、思考或傳聞的內容。它連接被引用的部分（被引用的內容）與引用動詞（如言う、思う、聞く等）。

**「と」的本質功能**：

1. **邊界標示**：區隔被引用的內容與引用的行為
2. **內容指示**：標示「と」前面的部分是引用內容
3. **形式保存**：直接引用時，保留原話的形式

**與引用動詞的搭配**：

引用的「と」必須搭配引用動詞使用。主要的引用動詞類別包括：

- **發言類**：言う（說）、話す（講）、答える（回答）、尋ねる（詢問）、叫ぶ（喊）、呼ぶ（叫）
- **思考類**：思う（想）、考える（考慮）、感じる（感覺）、信じる（相信）、決める（決定）
- **傳達類**：聞く（聽/問）、読む（讀）、書く（寫）、伝える（傳達）、知らせる（告知）
- **命名類**：呼ぶ（稱呼）、名付ける（命名）、いう（〜という名前）

引用的「と」與條件的「と」（如「雨が降ると中止だ」）形式相同，但可透過後續動詞的種類來區分。引用的「と」後面必定接續與發言、思考、傳達相關的動詞。

## 核心用法

### 用法 1：直接引用（原話引述）

將他人的話語原封不動地引用，使用引號括起來。

**例句 1**
```
先生は「静かにしなさい」と言いました。
The teacher said, "Be quiet."
老師說「安靜」。
```

**例句 2**
```
彼女は「もう帰ります」と言って、出て行った。
She said, "I'm going home now," and left.
她說「我要回去了」，然後離開了。
```

### 用法 2：間接引用（內容轉述）

用自己的話轉述他人說過的內容，使用普通形。

**例句 3**
```
田中さんは明日来ないと言っていました。
Tanaka-san said he wouldn't come tomorrow.
田中說他明天不會來。
```

**例句 4**
```
彼は自分が悪かったと認めた。
He admitted that he was wrong.
他承認是自己的錯。
```

### 用法 3：思考內容的表達

表達自己或他人心中所想的內容。

**例句 5**
```
私はこの計画がうまくいくと思います。
I think this plan will work well.
我認為這個計畫會順利進行。
```

**例句 6**
```
彼女は何も悪くないと信じている。
She believes that she is not at fault at all.
她相信自己完全沒有錯。
```

### 用法 4：疑問內容的引用

引用疑問句或詢問的內容。

**例句 7**
```
いつ届くのかと尋ねました。
I asked when it would arrive.
我問了什麼時候會送到。
```

**例句 8**
```
どうすればいいかと悩んでいる。
I'm wondering what I should do.
我正在煩惱該怎麼辦。
```

### 用法 5：命名與定義（という）

表達某事物的名稱或定義。

**例句 9**
```
「さくら」という名前の女の子がいます。
There is a girl named "Sakura."
有一個叫「櫻」的女孩。
```

**例句 10**
```
愛というものは言葉では説明できない。
The thing called love cannot be explained in words.
所謂的愛是無法用言語說明的。
```

## 文法規則

### 規則 1：引用部分使用普通形

間接引用時，被引用的部分必須使用普通形（常体），不使用敬體。

```
◯ 彼は忙しいと言った。
✗ 彼は忙しいですと言った。

He said he was busy.
他說他很忙。
```

### 規則 2：名詞・な形容詞的「だ」處理

名詞和な形容詞在現在肯定形時，「だ」可以省略（口語常見）或保留。

```
◯ 彼は学生だと言った。（保留だ）
◯ 彼は学生と言った。（省略だ，較口語）

He said he was a student.
他說他是學生。
```

但過去形、否定形不可省略：
```
◯ 彼は学生だったと言った。（過去）
◯ 彼は学生ではないと言った。（否定）
```

### 規則 3：疑問文的引用處理

引用疑問文時，可保留疑問詞，句末的「か」可加可不加。

**疑問詞疑問句**：
```
◯ どこに行くのかと聞いた。（加か）
◯ どこに行くと聞いた。（省略か，較口語）

I asked where (he) was going.
我問了要去哪裡。
```

**是非疑問句**：
```
◯ 明日来るかと聞いた。
◯ 明日来るかどうかと聞いた。

I asked if (he) would come tomorrow.
我問了明天會不會來。
```

### 規則 4：命令形的引用處理

命令形可直接引用，但語氣較強。通常會轉換為較婉轉的表達。

```
◯ 「早く来い」と言った。（直接命令，較粗魯）
◯ 早く来るようにと言った。（間接，較婉轉）
◯ 早く来てほしいと言った。（願望形式，更婉轉）

He told (me) to come quickly.
他叫（我）快點來。
```

## 常見錯誤

### 錯誤 1：間接引用使用敬體

❌ 誤：先生は明日は休みですと言いました。
✅ 正：先生は明日は休みだと言いました。
說明：間接引用必須使用普通形。「です」是敬體，應改為普通形「だ」。

```
先生は明日は休みだと言いました。
The teacher said tomorrow is a holiday.
老師說明天放假。
```

### 錯誤 2：遺漏引用動詞

❌ 誤：彼は来ないと。（句子不完整）
✅ 正：彼は来ないと言った。
說明：引用的「と」後面必須接續引用動詞，不能獨立使用。

```
彼は来ないと言った。
He said he wouldn't come.
他說他不會來。
```

### 錯誤 3：混淆引用「と」與條件「と」

❌ 誤：雨が降ると思います。（被誤解為條件）
✅ 正：雨が降ると思います。（正確理解為引用）
說明：這個句子實際上是正確的引用用法。「思う」是思考動詞，所以「と」是引用。如果是「雨が降ると、試合は中止です」則是條件用法。

```
引用：雨が降ると思います。（I think it will rain.）
條件：雨が降ると、試合は中止です。（If it rains, the game is canceled.）
```

### 錯誤 4：直接引用時省略引號

❌ 誤：彼は今すぐ来てと言った。（不清楚是直接還是間接）
✅ 正：彼は「今すぐ来て」と言った。（直接引用）
✅ 正：彼は今すぐ来てほしいと言った。（間接引用）
說明：直接引用應使用引號標示，間接引用應轉換為適當的表達形式。

```
直接：彼は「今すぐ来て」と言った。
He said, "Come right now."
他說「現在馬上來」。

間接：彼は今すぐ来てほしいと言った。
He said he wanted (me) to come right now.
他說希望（我）現在馬上來。
```

## 學習要點

1. **普通形原則**：間接引用部分必須使用普通形，這是最重要的規則
2. **引用動詞必須**：引用的「と」後面必須接續引用動詞（言う、思う、聞く等）
3. **直接vs間接**：直接引用保留原話並用引號，間接引用轉換為普通形
4. **だ的省略**：名詞和な形容詞的「だ」在口語中常省略，但過去形和否定形不可省
5. **與條件的區別**：根據後續動詞判斷是引用還是條件
